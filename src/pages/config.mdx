# 配置参考

SnowLuma 的 OneBot 配置采用 JSON 文件。首次为某个 QQ 会话启动 OneBot 实例时，会读取全局配置并生成账号级配置。

## 配置文件位置

```text
config/onebot.json
config/onebot_<uin>.json
```

- **全局配置**：`config/onebot.json`，作为所有账号的默认值。
- **账号配置**：`config/onebot_<uin>.json`，会覆盖全局配置。
- **自动生成**：缺少账号配置或访问令牌时，SnowLuma 会自动写入规范化后的配置。

## 默认配置

```json
{
  "httpServers": [
    {
      "host": "0.0.0.0",
      "port": 3000,
      "path": "/",
      "accessToken": "自动生成"
    }
  ],
  "httpPostEndpoints": [],
  "wsServers": [
    {
      "host": "0.0.0.0",
      "port": 3001,
      "path": "/",
      "role": "universal",
      "accessToken": "自动生成"
    }
  ],
  "wsClients": [],
  "musicSignUrl": ""
}
```

## httpServers

启动 OneBot HTTP API 服务。

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `name` | string | - | 端点名称。 |
| `host` | string | `0.0.0.0` | 监听地址。 |
| `port` | number | `3000` | 监听端口。 |
| `path` | string | `/` | API 路径前缀。 |
| `accessToken` | string | 自动生成 | OneBot 鉴权令牌。 |

## wsServers

启动 OneBot WebSocket 服务。

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `name` | string | - | 端点名称。 |
| `host` | string | `0.0.0.0` | 监听地址。 |
| `port` | number | `3001` | 监听端口。 |
| `path` | string | `/` | WebSocket 路径。 |
| `role` | string | `universal` | 可选 `api`、`event`、`universal`。 |
| `accessToken` | string | 自动生成 | 连接鉴权令牌。 |

## wsClients

SnowLuma 主动连接反向 WebSocket 服务。

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `name` | string | - | 端点名称。 |
| `url` | string | - | 反向 WebSocket 地址。 |
| `role` | string | `universal` | 可选 `api`、`event`、`universal`。 |
| `accessToken` | string | 自动生成 | 请求鉴权令牌。 |
| `reconnectIntervalMs` | number | `5000` | 重连间隔，最小会被规范化为 `1000`。 |

## httpPostEndpoints

将事件上报到 HTTP POST 端点。

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `name` | string | - | 端点名称。 |
| `url` | string | - | 上报地址。 |
| `accessToken` | string | - | 上报鉴权令牌。 |
| `timeoutMs` | number | - | 请求超时时间。 |

## 环境变量

| 变量 | 说明 |
| --- | --- |
| `SNOWLUMA_VERBOSE_WARMUP=1` | 输出更详细的好友、群、群成员预热日志。 |

## 端口冲突处理

如果 `3000` 或 `3001` 已被占用，请在账号配置中修改端口，然后重启 SnowLuma。
