<ActionHeader name={"get_group_ignore_add_request"} summary={"获取被忽略的入群请求（NapCat）"} readOnly={true} category={"扩展"} />

通过 `POST http://127.0.0.1:3000/get_group_ignore_add_request`（请求体为 JSON 参数）或 WebSocket 调用。

## 输入参数

该动作无需参数。

## 返回 (data)

被忽略的入群请求数组（NapCat 字段命名），每项含请求序列、邀请人、群信息与处理标记。

所有响应都包裹在标准信封中：`{ "status": "ok", "retcode": 0, "data": ... }`。下表描述 `data` 字段。

`data` 为数组，每个元素：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `request_id` | integer | ✓ | 请求序列号 |
| `invitor_uin` | integer | ✓ | 邀请人 QQ 号 |
| `invitor_nick` | string | ✓ | 邀请人昵称 |
| `group_id` | integer | ✓ | 群号 |
| `message` | string | ✓ | 验证留言 |
| `group_name` | string | ✓ | 群名称 |
| `checked` | boolean | ✓ | 是否已处理 |
| `actor` | integer | ✓ | 处理人 QQ 号 |
| `requester_nick` | string | ✓ | 申请人昵称 |
