<ActionHeader name={"get_group_ignore_add_request"} summary={"获取被忽略的入群请求（NapCat）"} readOnly={true} category={"扩展"} />

Call via `POST http://127.0.0.1:3000/get_group_ignore_add_request` (JSON params as the body) or WebSocket.

## Parameters

This action takes no parameters.

## Returns (data)

被忽略的入群请求数组（NapCat 字段命名），每项含请求序列、邀请人、群信息与处理标记。

Every response is wrapped in the standard envelope: `{ "status": "ok", "retcode": 0, "data": ... }`. The table below describes `data`.

`data` is an array; each element:

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `request_id` | integer | ✓ | 请求序列号 |
| `invitor_uin` | integer | ✓ | 邀请人 QQ 号 |
| `invitor_nick` | string | ✓ | 邀请人昵称 |
| `group_id` | integer | ✓ | 群号 |
| `message` | string | ✓ | 验证留言 |
| `group_name` | string | ✓ | 群名称 |
| `checked` | boolean | ✓ | 是否已处理 |
| `actor` | integer | ✓ | 处理人 QQ 号 |
| `requester_nick` | string | ✓ | 申请人昵称 |
